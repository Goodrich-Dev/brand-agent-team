# Personal-Brand Agent Team (Core)

Eleven installable Claude agents — ten in run order plus the operating-principles file — that take any expert from "accomplished but invisible" to a working personal-brand system: positioning, proof, an entity home, Knowledge Panel plumbing, AI-search visibility, $1/day distribution, and a content engine.

This is the **generic core**. Partner and vertical editions ("skins") derive from these files by tuning voice, audience, and funnel — the methodology stays identical. If you're building a skin, change the audience framing and the funnel specifics; never fork the method.

## The eleven, in run order

| # | Agent | What it does |
|---|---|---|
| 1 | `personal-brand-strategist` | Name your positioning: buy box, signature, one-sentence differentiation |
| 2 | `positive-mentions-harvester` | Harvest and rank every good thing already said about you |
| 3 | `reputation-gap-analyzer` | Score the gap between the reputation you have and need; dated 30-day plan |
| 4 | `personal-brand-website-agent` | Build your entity home — the facts-first site at your own name |
| 5 | `knowledge-panel-entity-seo` | Person schema, entity audit, the path to a Knowledge Panel |
| 6 | `ai-search-visibility` | Make ChatGPT/Perplexity/Google AI describe you correctly |
| 7 | `dollar-a-day-strategist` | $1/day boosts on proven content → opt-ins → entry-offer sales |
| 8 | `content-factory` | One recording in, a week of on-voice content out |
| 9 | `definitive-article-writer` | The one canonical page that owns your name or topic |
| 10 | `recursive-self-improvement-qa` | Every agent grades and improves its own work before you see it |
| — | `boil-the-ocean` | Operating principles — install alongside the ten |

## Install

1. **Paste it** — open any `skills/<agent>/SKILL.md`, copy, paste into Claude with "Follow this. Here are my details: …"
2. **Install as a Skill** — Settings → Capabilities → Skills → upload.
3. **Project / memory** — drop the files into a Claude Project's knowledge.

Every agent starts by reading your **brand-voice document** and **ideal-client context file** so output sounds like you. Build those first.

## Skins

Partner editions live in their own repos (e.g. the SOMBA edition for Sigrun's program). A skin changes: the audience framing, the funnel specifics (lead magnet / entry offer naming, currency), the read-first file references, and the notes sections. It must not change: steps, definitions of done, or the run order.
